# DailyBCI 产品蓝图

> 2026-06-04 头脑风暴定稿

---

## 一、产品定义

**一句话描述**：每日精选 1-2 条 BCI 领域最重要的学术进展，用清晰的语言讲明白"他们做了什么、为什么重要"，中英双版，分发至 X 和小红书。

**核心价值**：不是新闻聚合，而是"学术解读"——把复杂的 BCI 论文和会议成果翻译成任何研究者都能在 3 分钟内理解的内容。

**品牌身份**：BCI 学术解读者。不以专家自居，以"比任何单个研究者看得更广、更快、更跨子领域"的视角提供价值。使用笔名/品牌名运营。

**命名**：待定（建议发布几期找到声音后再确定）。

---

## 二、目标受众

**主要受众**：BCI 领域学术研究者（博士生、博后、PI），全子领域覆盖。

**平台特征**：
- X/Twitter（英文版）：直接触达全球 BCI 学术社区
- 小红书（中文版）：触达中国高校/研究机构的硕博和青年研究者

## 三、内容策略

### 3.1 每期规模

- **严格 1-2 条**，说透即可
- 没有大事的日子降级选题但不断更
- 不追求信息全覆盖，追求选题精准

### 3.2 内容模板（三层结构，比例 7:2:1）

| 层次 | 占比 | 内容 |
|------|------|------|
| **他们做了什么** | ~70% | 用简明语言解读核心方法、实验设计、关键结果 |
| **为什么重要** | ~20% | 这件事在领域中的意义，解决了什么问题 |
| **我的一点思考** | ~10% | 简短的个人视角、联想、或一个好问题 |

大头是解读，把复杂内容讲清楚讲明白。个人观点只是附带的小尾巴，不强求鲜明立场。

### 3.3 信源优先级

| 优先级 | 信源类型 | 说明 |
|--------|---------|------|
| **第一优先** | 学术预印本 & 会议 | arXiv、bioRxiv、medRxiv、SfN、NeurIPS、BCI Society 等 |
| 第二优先 | 社交媒体 | X/Twitter 上领域内研究者的讨论和分享 |
| 第三优先 | 公司动态 | Neuralink、Synchron、Paradromics 等公司新闻 |
| 第四优先 | 监管与资助 | FDA、NIH BRAIN Initiative、ClinicalTrials.gov |

会议季（SfN、NeurIPS 等）期间做专题覆盖。

### 3.4 子领域范围

**全覆盖**，不做子领域限制：
侵入式、非侵入式、运动解码、语音解码、感觉反馈、神经调控、电极技术、信号处理、闭环系统、临床试验、脑机接口算法/AI……

### 3.5 筛选标准

"值得发"的门槛包括但不限于：
- 方法论突破（新的解码方法、电极技术、信号处理思路）
- 临床/转化里程碑（first-in-human、FDA 批准、患者数据）
- 刷新记录的结果（解码速度、精度、通道数等量化指标新高）
- 其他对领域有实质意义的进展

### 3.6 语气风格

**专业但不刻板**——像和同行在咖啡厅聊天，有术语但不僵硬。两个平台保持一致的语气。

## 四、双语 & 双平台策略

### 4.1 语言

- 同一内容出中英两个版本
- 信息和观点保持一致，但表现形式适配各自平台
- 不是逐字翻译，而是同一信息的两种语言表达

### 4.2 平台适配

| 维度 | X (英文) | 小红书 (中文) |
|------|---------|-------------|
| 格式 | 每日一条长 thread | 图文帖（简洁配图 + 文字） |
| 链接 | 附原始论文/新闻链接 | 附原始源链接 |
| 视觉 | 纯文本为主 | 需要图片（待定视觉方案） |
| 深度入口 | thread 内直接链接原始源 | 正文/评论区附链接 |

### 4.3 小红书视觉方案

**待定**。初期可以用统一模板的文字卡片，稳定后再优化。核心原则：信息本身是价值，不靠花哨设计。

## 五、技术架构

### 5.1 核心创新：BCI 知识库

**这是产品最大的技术壁垒和差异化资产。**

- **性质**：200-500 篇 BCI 领域 milestone papers 的结构化摘要库
- **构建方法**：用权威综述论文（review papers）作为种子，提取其引用的里程碑工作
- **存储形式**：结构化 markdown 文件，按子领域分类存放
- **使用方式**：当 AI 评价新论文时，根据主题自动加载最相关的历史论文摘要作为上下文（"穷人版 RAG"——目录结构 + 关键词匹配）
- **迭代机制**：自动积累（每天发布的重要论文自动加入）+ 定期人工筛选和清理

### 5.2 技术约束

- **不使用 API**，全部在 Claude Pro / ChatGPT Plus 的对话框内完成
- **实现形式**：Codex skill 或 Claude skill，在对话中调用
- **复用现有管线**：dailyBCI 文件夹中已有的 Python 脚本（RSS 抓取、网页抓取、关键词过滤、评分）

### 5.3 每日工作流

```
[自动] 信源采集 → 关键词过滤 → 相关性评分 → Top 候选
                         ↓
[自动] 加载知识库上下文 → AI 生成初稿解读
                         ↓
[人工] 你和 AI 对话 2-3 轮打磨内容（~30 分钟）
                         ↓
[自动] 生成英文 X thread + 中文小红书图文（允许微调）
                         ↓
[人工] 你手动发布到两个平台
```

### 5.4 缺席模式

当你不在时（生病/旅行/忙），允许 AI 全自动生成并发布，跳过人工审核环节。

## 六、运营策略

### 6.1 更新频率

**每日更新**，早上固定时间发布。没有休息日。

### 6.2 冷启动

- **零基础起步**，没有现有粉丝或学术人脉
- **X 增长策略**：评论/引用领域内大号（Sergey Stavisky、BrainGate、Neuralink 官方等），让解读出现在他们的讨论区
- **小红书增长策略**：靠平台算法推荐，优化关键词和封面图

### 6.3 成功指标

**首要指标：自我成长**——通过每天做这件事加深对 BCI 领域的理解。

粉丝数量和行业认可是自然的副产品，不是首要目标。

### 6.4 长期演进

解读者 → 评论员：随着每天的积累，逐步建立自己的判断框架，从"讲清楚别人做了什么"进化到"对领域发展有自己的判断和预测"。

## 七、启动计划

### Phase 0：知识库建设（1-2 周）

1. 收集 5-10 篇权威 BCI 综述论文
2. 提取其中引用的 milestone papers，建立初始 30-50 篇的核心知识库
3. 每篇整理为结构化摘要（方法、问题、贡献、子领域标签）
4. 按子领域分类存放

### Phase 1：试运行（第 3 周起）

1. 用现有数据管线 + 小规模知识库开始每日生成内容
2. 手动对话打磨 + 手动发布
3. 边跑边优化选题标准和内容质量
4. 同步扩充知识库

### Phase 2：稳定运营

1. 知识库扩充至 200-500 篇
2. 内容生成流程半自动化（自动生成初稿和图片，人工微调后发布）
3. 确定品牌名称和视觉方案

### Phase 3：全自动化

1. 信源采集 → 筛选 → 内容生成 → 格式适配全链路自动化
2. 你只做最终审核、深度对话、和手动发布
3. 缺席时可切换为全自动模式
